## 什么是事务

事务是控制数据库操作的一段逻辑，要么全部成功，要么全部失败



具有`ACID`的特性



`A`：原子性，要么全部成功，要么全部失败

`C`：一致性，保证事务前后数据相加一致

`I`: 隔离性，多个并发事务执行中互不干扰，也与数据库的隔离级别有关

`D`：持久性，事务完成后数据被写入硬盘持久化
